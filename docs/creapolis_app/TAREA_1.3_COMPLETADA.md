## üéì Aprendizajes

### Decisiones de Dise√±o

1. **Agrupaci√≥n Temporal vs Lista Flat**

   - **Elegido:** Agrupaci√≥n temporal
   - **Raz√≥n:** Organiza visualmente las tareas por urgencia
   - **Beneficio:** Usuario identifica r√°pidamente qu√© hacer hoy vs pr√≥ximamente

2. **Swipe Actions vs Men√∫ Contextual**

   - **Elegido:** Swipe Actions (Dismissible)
   - **Raz√≥n:** Interacci√≥n m√°s moderna y r√°pida
   - **Beneficio:** Completar/eliminar tareas con un gesto natural

3. **Quick Complete Button vs Solo Swipe**

   - **Elegido:** Ambos (Quick Button + Swipe)
   - **Raz√≥n:** Dos formas de lograr lo mismo ‚Üí m√°s flexible
   - **Beneficio:** Usuarios con diferentes preferencias est√°n cubiertos

4. **TabBar vs Dropdown**

   - **Elegido:** TabBar
   - **Raz√≥n:** M√°s est√°ndar en apps m√≥viles, cambio visual m√°s r√°pido
   - **Beneficio:** Usuario ve claramente las dos opciones sin abrir men√∫

5. **Ordenamiento con Toggle vs Dos Botones**
   - **Elegido:** Toggle (mismo bot√≥n invierte direcci√≥n)
   - **Raz√≥n:** Menos clutter en AppBar
   - **Beneficio:** √çcono din√°mico indica direcci√≥n actual

### Desaf√≠os Resueltos

1. **Agrupaci√≥n de Tareas por Fecha**

   - **Problema:** C√°lculo complejo de "esta semana" considerando d√≠a de la semana
   - **Soluci√≥n:** `DateTime(now.year, now.month, now.day)` normaliza hora, luego calcular d√≠as hasta fin de semana
   - **Aprendizaje:** Siempre normalizar fechas antes de comparar

2. **Confirmaci√≥n en Dismissible**

   - **Problema:** Dismissible elimina inmediatamente, no permite confirmaci√≥n
   - **Soluci√≥n:** Usar `confirmDismiss` callback que retorna `Future<bool>`
   - **Aprendizaje:** `confirmDismiss` permite validaciones as√≠ncronas antes de dismiss

3. **Estado Independiente por Tab**

   - **Problema:** Filtros se compart√≠an entre tabs
   - **Soluci√≥n:** Pasar `myTasksOnly` como par√°metro a `_buildContent()`
   - **Aprendizaje:** Estado global (`_filterStatus`) + par√°metros locales (`myTasksOnly`) = flexibilidad

4. **Ordenamiento Bidireccional**

   - **Problema:** Usuario quiere ver tanto fecha m√°s reciente primero como m√°s antigua
   - **Soluci√≥n:** Variable `_sortAscending` + l√≥gica de toggle si ya est√° en ese criterio
   - **Aprendizaje:** Comparaci√≥n con ternario: `_sortAscending ? comparison : -comparison`

5. **Contador de Filtros Activos**
   - **Problema:** Usuario pierde track de cu√°ntos filtros tiene aplicados
   - **Soluci√≥n:** Badge en IconButton con `Positioned` y contador calculado
   - **Aprendizaje:** UI reactiva mejora awareness del usuario

### Patrones Implementados
